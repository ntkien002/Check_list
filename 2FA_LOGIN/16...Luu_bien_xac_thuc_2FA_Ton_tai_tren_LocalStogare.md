Bài write-up trong link mô tả một **logic flaw trong triển khai 2FA**: server tin tưởng dữ liệu phía client (cookie/localStorage). Vì vậy attacker có thể **tự tạo giá trị xác nhận 2FA** mà không cần nhập OTP. ([Medium][1])

Dưới đây là phân tích kỹ thuật.

---

# 1. Bản chất lỗ hổng

Ứng dụng sử dụng một biến phía client:

```
isVerifyAuth
```

Biến này lưu trong **localStorage / cookie** để đánh dấu rằng user đã vượt qua bước OTP.

Flow logic của hệ thống:

```
Login (username + password)
        ↓
Redirect -> /verify-otp
        ↓
Nếu isVerifyAuth == true
        ↓
Cho phép truy cập dashboard
```

Vấn đề:
Server **không tự kiểm tra OTP**, mà chỉ **tin vào giá trị cookie/localStorage**.

Do cookie/localStorage **hoàn toàn do client kiểm soát**, attacker có thể sửa hoặc tạo mới.

---

# 2. PoC bypass

Quy trình khai thác rất đơn giản.

### Bước 1

Login bình thường

```
POST /login
```

Sau đó hệ thống redirect:

```
/customer/verify-otp
```

---

### Bước 2

Mở DevTools → Application → Local Storage

Tạo key mới:

```
isVerifyAuth = true
```

---

### Bước 3

Truy cập trực tiếp page sau login

```
/customer/profile
```

Kết quả:

```
Access granted
```

Không cần OTP. ([Medium][1])
